# AQIndia — Dataset Documentation

This document describes all datasets used in the AQIndia project for reproducibility and transparency.

## 📊 Primary Datasets

### 1. Kaggle Air Quality Data in India (2015-2024)

**Source:** Kaggle Dataset - Air Quality Data in India

**Description:**
- Comprehensive air quality measurements from Central Pollution Control Board (CPCB) stations
- Covers 108+ Indian cities across all major states
- Time period: 2015 to 2024 (10 years of data)
- Contains both raw measurements and calculated AQI values

**File in Project:**
- Original CSV: `jupyter-notebooks v2/datasets/city_day_comprehensive_2026.csv` (28 MB)
- Processed JSON: `aqindia/server/data/kaggle_historical.json` (3.4 MB)
- Raw JSON (backup): `aqindia/server/data/kaggle_historical_original.json` (87 MB) ⚠️ *Not pushed to GitHub due to size*

**Columns:**
- `City` - City name
- `Datetime` - Date and time of measurement
- `AQI` - Air Quality Index (calculated)
- `PM2.5` - Particulate Matter 2.5µm (µg/m³)
- `PM10` - Particulate Matter 10µm (µg/m³)
- `NO2` - Nitrogen Dioxide (µg/m³)
- `SO2` - Sulfur Dioxide (µg/m³)
- `CO` - Carbon Monoxide (mg/m³)
- `NH3` - Ammonia (µg/m³)
- `O3` - Ozone (µg/m³)

**Data Quality:**
- Contains missing values (NaN) - typical of real-world environmental data
- Some cities have more complete data than others
- Requires cleaning and imputation before ML training

**Usage in Project:**
1. Loaded in Jupyter Notebook 01 for exploration
2. Cleaned and preprocessed in Notebooks 01-04
3. Used to train ML models in Notebooks 05-07
4. Converted to JSON format for use in React dashboard
5. Powers all historical charts and analytics

**Note:** This dataset was manually selected and curated for this project from publicly available air quality data sources.

**Statistics:**
- Total records: ~1,000,000+ daily measurements
- Cities covered: 108+
- Time span: 2015-2024
- Missing values: ~15-30% depending on pollutant and city

### 2. Train/Test Splits (ML Ready)

**Description:**
Pre-processed and feature-engineered datasets ready for machine learning training.

**Files:**
- `X_train.csv` (142 MB) - Training features
- `X_test.csv` (35 MB) - Test features
- `y_train_reg.csv` (2.2 MB) - Training labels (regression)
- `y_test_reg.csv` (0.55 MB) - Test labels (regression)
- `y_train_clf.csv` (1 MB) - Training labels (classification)
- `y_test_clf.csv` (0.25 MB) - Test labels (classification)

⚠️ *Not pushed to GitHub due to size. Generated from Kaggle dataset.*

**Features Engineered:**
- Temporal features: day_of_week, month, season, is_weekend
- Lag features: pm25_lag_1, pm25_lag_7, pm25_lag_30
- Rolling statistics: pm25_rolling_mean_7, pm25_rolling_std_7
- Meteorological features: temperature, humidity, wind_speed
- Interaction features: pm25_x_temperature, pm10_x_humidity
- Total: 60+ engineered features

**Usage:**
- Used to train XGBoost, Random Forest, LSTM, Prophet models
- Models saved in `jupyter-notebooks v2/models/`
- Model metrics stored in `aqindia/server/data/ml_metrics.json`

### 3. Real-Time AQI Data (Open-Meteo API)

**Source:** Open-Meteo Air Quality API (Free, no API key required)

**Description:**
- Free, no API key required
- Global coverage (1000+ cities)
- Real-time hourly measurements
- European AQI scale (converted to Indian AQI in app)

**Endpoint:**
```
GET https://api-v2.open-meteo.com/v1/air-quality
?latitude=28.7041
&longitude=77.1025
&hourly=pm10,pm2_5,no2,so2,o3,co
&timezone=Asia/Kolkata
```

**Usage in Project:**
- Powers live AQI dashboard on home page
- Updated every 5 minutes (cached)
- Fallback to WAQI API if user provides API key

**File in Project:**
- `aqindia/server/data/current_aqi.json` (63 KB) - Latest snapshot

## 🔬 Data Quality Notes

### Known Issues in Original Dataset:
1. **Missing Values:** ~15-30% of pollutant readings are NaN
   - Handled by: Forward fill, backward fill, seasonal averages
2. **Outliers:** Some extreme AQI values (>700)
   - Handled by: Clamping to 0-500 range (US EPA scale)
3. **Inconsistent City Names:** Same city with different spellings
   - Handled by: Standardization mapping
4. **Temporal Gaps:** Some cities have missing months/years
   - Handled by: Interpolation where possible

### Data Cleaning Steps Applied:
1. Removed rows with all NaN values
2. Forward fill for short gaps (<7 days)
3. Seasonal average fill for longer gaps
4. Outlier detection using IQR method
5. AQI range validation (0-500)
6. Date sorting and duplicate removal
